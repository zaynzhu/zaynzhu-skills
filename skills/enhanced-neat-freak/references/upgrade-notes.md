# Enhanced Neat Freak 升级说明

本文说明 `enhanced-neat-freak` 相对本地旧版 `neat-freak` 的增强方向，供维护技能时参考。

## 旧版已经做得好的部分

- 明确区分 agent 记忆、项目级指令、README/docs 三类知识层。
- 强调 `CLAUDE.md` / `AGENTS.md` 是规则手册，不是变更日志。
- 通过尺寸体检防止项目级指令无限膨胀。
- 有跨平台路径参考和变更影响矩阵。
- 要求真实编辑文件，不只给建议。

## 主要增强点

### 1. 模式化执行

旧版默认进入完整同步流程，适合阶段收尾，但在用户只想"看看文档有没有问题"时容易过重。新版增加五种模式：

- `audit-only`：只审计，不改文件。
- `sync`：默认模式，做最小必要同步。
- `handoff`：以交接为目标，强调新人可读性。
- `anti-bloat`：优先精简和反膨胀。
- `memory-note`：只在用户明确要求时处理记忆。

### 2. 证据账本

旧版强调枚举文件，但没有强制记录每条判断的证据。新版要求先建立证据账本，把事实、来源、影响层、动作和验证方式写清楚。这样可以减少"凭印象更新文档"。

### 3. 平台规则优先

不同 agent 对记忆写入的权限不同。新版明确：如果上级规则或平台限制不允许自动写记忆，只输出建议文本，不绕过权限。

### 4. 最小同步优先

旧版容易被理解为"会话结束要全量整理所有文档"。新版先界定范围，只处理与本次请求或证据冲突直接相关的文件。无关坏味道列为风险，不顺手重构。

### 5. 验证成为交付条件

新版把验证拆成 Markdown 结构、事实搜索、git 范围、命令可运行性和记忆建议检查。不能验证时必须说明原因。

## 不新增的能力

- 不做磁盘清理、代码格式化或依赖升级。
- 不自动发布文档站。
- 不替代完整技术写作技能；它只负责同步、精简和交接。
- 不在没有证据时重写历史文档。

## 推荐后续迭代

- 给常见仓库类型增加轻量检查脚本，例如 Markdown 链接检查。
- 增加 `handoff.md` 模板，用于大型项目交接。
- 做描述触发评测，区分"整理文件夹"和"整理项目知识库"。
