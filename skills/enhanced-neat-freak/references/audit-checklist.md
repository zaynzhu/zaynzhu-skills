# 交付前自审清单

在输出最终同步结果前逐项检查。任何一项失败，都先修复或在风险项中明确说明。

## 范围控制

- [ ] 已确认项目根目录。
- [ ] 已查看 `git status --short`，知道当前是否有用户未提交改动。
- [ ] 只处理与本次请求、代码证据或文档冲突直接相关的文件。
- [ ] 无关历史坏味道没有被顺手重构。
- [ ] 如果跨项目范围不清楚，已列为风险而不是猜测修改。

## 证据

- [ ] 每条新增或删除的信息都有证据来源。
- [ ] 路径、命令、环境变量、API 名称已用搜索或文件读取验证。
- [ ] 没有把记忆中的旧事实当作当前事实直接写入。
- [ ] 发现证据冲突时已标为需要用户确认。

## 反膨胀

- [ ] `CLAUDE.md` / `AGENTS.md` 没有新增历史叙事。
- [ ] README 没有塞入内部机制全文。
- [ ] docs 没有重复已有权威说明。
- [ ] 新增文字尽量合并到现有段落，而不是总在顶部追加。
- [ ] 删除的内容确实过期、重复或不属于该层级。

## 记忆处理

- [ ] 只有在用户明确要求且平台规则允许时，才写入记忆。
- [ ] 记忆建议不含敏感信息。
- [ ] 记忆建议不含"今天/最近/刚刚"等相对时间。
- [ ] 一条记忆只表达一件可复用事实。

## 验证

- [ ] 运行了最小可行验证命令，或说明无法运行的原因。
- [ ] `git diff --check` 没有空白错误。
- [ ] 新引用的 `references/`、`scripts/`、`assets/` 文件都存在。
- [ ] Markdown 链接或路径没有明显拼写错误。
- [ ] 交付说明包含：修改了什么、为什么、如何验证、风险。

## 输出洁净度

- [ ] 最终回复只列实际变更，不罗列未动文件。
- [ ] 风险项具体到路径、关键词或下一步。
- [ ] 没有承诺未执行的动作。
- [ ] 若是 `audit-only`，明确说明没有改文件。
