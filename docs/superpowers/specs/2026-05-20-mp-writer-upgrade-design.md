# mp-writer 全面升级设计文档

> 日期：2026-05-20
> 状态：待审阅

## 背景

mp-writer 是一个公众号长文写作 skill，当前包含完整的风格体系、写作流程和四层自检体系。经过全面审阅，发现以下核心问题：

1. **流程效率低**：没有大纲步骤，AI 边写边想导致结构散乱；L1 硬性规则应该是写的时候就遵守，而不是写完再扫
2. **风格规则太散**：20+ 条风格规则平铺在一起，AI 不知道哪些必须做到，哪些选择性使用，结果什么都想沾一点，哪条都没用到位
3. **适用范围窄**：只支持长文，不支持短内容（小红书/朋友圈）、续写、改写
4. **热点搜索过于泛化**：不同文章类型需要搜索的东西完全不同，但当前没有区分

## 改进方案

### 一、流程重构

**旧流程**：
```
素材 → 热点搜索 → 选题判断 → (直接写) → 四层自检 → 修改 → 再检 → 输出
```

**新流程**：
```
素材 → 热点搜索 → 选题判断 → 大纲生成 → 用户确认大纲 → 写作 → 快扫+终审 → 输出
```

关键变化：

1. **新增大纲步骤**：在正式写作前生成文章骨架，包含：
   - 文章原型判断
   - 开头方式选择
   - 每段核心观点 + 素材分配
   - 选定的风格锦囊（3-5 个）
   - 文化升维锚点
   - 结尾类型
   - 回环呼应设计（开头埋什么钩子，结尾怎么 callback）

2. **L1 规则内置到写作指令**：禁用词、禁用标点、套话模式写进"写作时的绝对约束"区，写的时候就遵守

3. **自检简化为快扫 + 终审**：详见第四部分

### 二、风格体系分层

从 20+ 条平铺规则变为三层结构：

#### 核心锚点（每篇必须做到）

1. **活人语气**：像跟朋友聊天。句子时长时短，大量逗号制造口语停顿。一句话独立成段的断裂效果全文至少 3 次。
2. **具体不空泛**：每个核心观点都有具体的人/场景/细节/数据支撑。不说"AI工具"，说具体名字。不用假设性例子，用"我今天正在搞的"真实细节。
3. **绝对禁区零容忍**：禁用词、禁用标点、套话模式、教科书开头——硬性红线，写的时候就必须遵守。

#### 风格锦囊（根据文章类型选择 3-5 个）

不同文章类型的锦囊侧重：

| 文章类型 | 推荐锦囊 |
|---------|----------|
| 调查实验型 | 亲自下场、英雄之旅叙事弧、逐一展示法 |
| 产品体验型 | 逐一展示法、创意案例、荒诞比喻 |
| 现象解读型 | 文化升维、层层剥开、对立面理解 |
| 工具分享型 | 个人故事铺垫、谦逊铺垫法 |
| 方法论分享型 | 谦逊铺垫法、坦诚学习曲线、每节可执行行动 |

AI 在大纲步骤中选定用哪 3-5 个锦囊，写作时专注用好选定的几个。

#### 质感调节（自然流露即可）

回环呼应、读者直呼、疑问句节奏、情绪标点（。。。？？？= =）。写着写着自然会用到，终审时如果完全没有可以适当补充。

### 三、多模式扩展

#### 四种模式

| 模式 | 触发词 | 字数 | 流程差异 |
|------|--------|------|---------|
| 长文模式 | 写文章、写稿子、公众号长文、出稿 | 4000-8000 | 完整五步流程 |
| 短内容模式 | 小红书、朋友圈、短推文、短文案 | 200-800 | 砍掉大纲和自检，只保留核心锚点 + 口语化词组 |
| 续写模式 | 续写、接着写、扩写、补全 | 续写部分 1000-3000 | 先风格对齐，再按长文流程续写 |
| 改写模式 | 用我的风格写、改写、改成我的风格、重写 | 看原文 | 提取原文骨架，按长文流程重建 |

#### 短内容模式细节

- **砍掉**：大纲步骤、四层自检、文化升维、人物画像法、英雄之旅叙事弧
- **保留**：核心锚点（活人语气、具体不空泛、绝对禁区）、口语化词组、开头必杀技中的"好奇心驱动"和"荒诞事实"
- **新增**：短内容结构模板（钩子 → 核心观点 → 行动呼吁 / 金句收尾）

#### 续写模式细节

1. 先读已有文本，提取风格指纹：节奏（长短句比例）、人称、标点习惯、情绪基调
2. 大纲步骤中，续写部分必须延续已有文本的风格指纹
3. 自检增加：风格衔接检查（续写部分和原文是否风格一致）

#### 改写模式细节

1. 分析原文：提取核心观点、关键素材、数据
2. 丢弃原文的语言风格
3. 按长文流程重新写，但素材来自原文

### 四、自检体系重构

从四层自检简化为快扫 + 终审。

#### 快扫（替代 L1+L2）

| # | 检查项 | 通过标准 |
|---|--------|---------|
| 1 | 禁用词 | 0 命中 |
| 2 | 禁用标点 | 0 命中（冒号、破折号、双引号） |
| 3 | 套话模式 | 0 命中 |
| 4 | 空泛工具名 | 0 命中 |
| 5 | 开头是否具体切入 | 不是宏大叙事开头 |
| 6 | 长短句交替 | 连续 3 句以上句式长度相近 = 不通过 |
| 7 | 口语化词组 | 至少 8 个不同表达 |
| 8 | 一句话独立成段 | 全文至少 3 次 |
| 9 | 扣主线句 | 每个偏离主线的段落后都有 |
| 10 | 小标题 | 除方法论型外不应出现 |

1-4 项命中即替换，5-10 项不通过就改写对应段落。

#### 终审（替代 L3+L4）

以读者视角通读全文，回答核心问题：

> **"读完这篇文章，我感觉是一个有见识的普通人在认真跟我聊一件打动他的事，还是一个 AI 在给我输出信息？"**

检查维度：
1. 每个核心观点都有具体支撑吗？（人/场景/细节/数据）
2. 知识是"随手掏出来"的还是"下面我来科普"的？
3. 有没有文化升维？连接是否自然？
4. 对立面有理解和承认吗？
5. 文章类型专项检查通过吗？
6. 活人感判断：温度感、独特性、姿态、心流——有 AI 味重的段落吗？

如果任何维度觉得"AI 味太重了"，就重写那个段落。

#### 不同模式的自检标准

| 模式 | 自检 |
|------|------|
| 长文 | 快扫 + 终审（完整） |
| 短内容 | 快扫 1-7 项 + 终审简化版（活人感+具体性） |
| 续写 | 快扫 + 终审 + 风格衔接检查 |
| 改写 | 快扫 + 终审（完整） |

### 五、热点搜索分类型

不同文章类型搜索不同维度的信息：

| 文章类型 | 搜索重点 |
|---------|---------|
| 调查实验型 | 事件时间线、当事人信息、数据统计、对比实验结果 |
| 产品体验型 | 产品最新动态、竞品对比、用户真实评价、价格信息 |
| 现象解读型 | 现象起源、不同立场观点、学术研究、历史类比 |
| 工具分享型 | 工具最新版本、使用教程、用户反馈、替代品 |
| 方法论分享型 | 行业最新实践、学术支撑、常见失败案例、数据佐证 |

## 文件结构

升级后的文件结构：

```
skills/mp-writer/
├── SKILL.md                          # 主文件：流程、模式、风格体系
├── references/
│   ├── style_examples.md             # 风格示例库（保留，微调）
│   ├── content_methodology.md        # 内容方法论（保留）
│   ├── mode-templates.md             # 新增：各模式结构模板和自检标准
│   └── search-guide.md               # 新增：分类型搜索指南
```

## 不变的部分

- 核心价值观（好奇、活人感、真诚、有所为有所不为）
- 风格内核的描述（节奏感、论述打破、知识输出方式等）
- 口语化词组列表
- 开头必杀技
- 文章原型分类
- references/ 下的示例库
- AI 角色边界（AI 擅长 vs 必须人来）
- 语言风格要求（中文为主、避免中英混杂）
